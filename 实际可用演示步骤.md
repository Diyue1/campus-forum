## 演示 4：测试跟踪与故障管理 ✅

### 可以做的演示

#### 4.1 展示测试跟踪脚本
```bash
code scripts\test-tracker.js
```
**讲解**：自己开发的测试跟踪系统，管理测试结果和 Bug。

#### 4.2 创建测试记录
```bash
node scripts\test-tracker.js record
```
**讲解**：记录测试运行结果。

#### 4.3 创建 Bug
```bash
node scripts\test-tracker.js bug create "演示Bug" "这是一个演示用的Bug" medium
```
**讲解**：创建 Bug，系统会自动分配 ID。

#### 4.4 查看 Bug 列表
```bash
node scripts\test-tracker.js bug list
```
**讲解**：查看所有 Bug 及其状态。

#### 4.5 更新 Bug 状态
```bash
# 使用上一步创建的 Bug ID
node scripts\test-tracker.js bug update BUG-XXXXX resolved "演示完成"
```
**讲解**：更新 Bug 状态为已解决。

#### 4.6 生成报告
```bash
node scripts\test-tracker.js report
```
**讲解**：生成测试跟踪报告，包括统计信息。

#### 4.7 查看报告
```bash
# 打开报告目录
explorer reports\test-tracking
```
**讲解**：查看生成的 HTML 和 JSON 报告。

```bash
git add test-demo.txt

# 尝试错误格式（会被拒绝）

```
**讲解**：不符合规范的提交信息会被拒绝。

```bash
# 使用正确格式
git commit -m "docs: 添加演示文件"
```
**讲解**：正确的格式：类型(范围): 描述。

#### 5.5 清理演示文件

## 🎯 推荐的演示流程

### 方案 A：完全离线演示（最安全）

1. **展示配置和代码**（5分钟）
   - 打开各种配置文件
   - 解释架构和设计

2. **运行本地测试**（10分钟）
   - `npm test`
   - `npm run test:coverage`
   - `npm run lint`

3. **演示测试跟踪系统**（5分钟）
   - 创建 Bug
   - 更新状态
   - 生成报告

4. **演示 Git Hooks**（5分钟）
   - 展示配置
   - 模拟提交验证

5. **展示文档和报告**（5分钟）
   - 打开之前生成的报告
   - 展示文档结构
